# Claude Code – Projektkontext & Arbeitsregeln

## Projekt

SEO Operating System – Operatives Handbuch für den SEO-Content-Fulfillment-Prozess (Amazon-Listings).

## Sprache

- Alle Inhalte, Commit-Messages und Kommunikation auf **Deutsch**
- Fachbegriffe (SEO, Keywords, Content, QA) dürfen auf Englisch bleiben

## Repository-Struktur

```
prozesse/        → IST-Prozesse und Workflows
optimierung/     → Massnahmenpläne und Verbesserungen
checklisten/     → QA-Scorecards und Onboarding-Checklisten
```

## Branch-Strategie

- Arbeite immer auf `claude/*` Feature-Branches
- Nie direkt auf `main` pushen
- Änderungen gehen per Pull Request in `main`

## Arbeitsregeln

- Markdown-Dateien (.md) als Hauptformat
- Bestehende Dateien erweitern statt neue anlegen, wenn thematisch passend
- Tabellen und Prozessdiagramme (ASCII) bevorzugen für Übersichtlichkeit
- Checklisten mit Checkboxen (`- [ ]`) für actionable Items

## Kontext

- Zielgruppe: Internes Team (Onboarding, Prozessoptimierung)
- Volumen: 5–100+ SEO-Contents pro Monat
- Tools im Einsatz: Helium 10, ChatGPT, Google Drive, Asana, Amazon Seller Central

# Kunden-Onboarding-Checkliste

Diese Checkliste stellt sicher, dass alle nötigen Informationen vorliegen, bevor die Content-Produktion startet. **Kein Asana-Ticket wird erstellt, solange die Pflichtfelder nicht vollständig sind.**

---

## Vor der Übergabe (Sales → Fulfillment)

### Pflicht-Inputs (Blocker)

- [ ] **Kundenname** und Ansprechpartner
- [ ] **E-Mail-Adresse** des Ansprechpartners
- [ ] **Markenname** (exakte Schreibweise wie auf Amazon)
- [ ] **Zielmarktplatz** (z.B. Amazon.de, Amazon.com)
- [ ] **Produktname(n)** und ggf. ASIN(s) bestehender Listings
- [ ] **Produktkategorie** auf Amazon
- [ ] **Produktinformationen** mindestens eines der folgenden:
  - [ ] Produktdatenblatt / technische Spezifikationen
  - [ ] Produktbeschreibung vom Hersteller
  - [ ] Bestehender Amazon-Listing-Text
- [ ] **Produktbilder** (min. 3 Bilder, um Produkt zu verstehen)
- [ ] **UVP / Preisspanne**
- [ ] **Scope** – Welche Leistungen wurden verkauft? (SEO-Text, Design, PPC etc.)
- [ ] **Google Drive Shared Folder** ist eingerichtet mit Template-Ordnerstruktur

### Optionale Inputs (beschleunigen den Prozess)

- [ ] **Voice & Tone** – Wie kommuniziert die Marke? (formell/locker, Zielgruppe, Sprache)
- [ ] **Wettbewerber-ASINs** – Kennt der Kunde seine Top-Wettbewerber?
- [ ] **Hauptkeywords** – Hat der Kunde bereits Keywords identifiziert?
- [ ] **Bestehende Listings** – Falls vorhanden, zum Vergleich/Verbesserung
- [ ] **Besondere Hinweise** – Rechtliche Einschränkungen, verbotene Claims, Zertifizierungen
- [ ] **Marken-Styleguide** – Farben, Schriftarten, Logo-Dateien

---

## Bei der Übergabe (Fulfillment-Setup)

### Google Drive prüfen

- [ ] Template-Ordnerstruktur ist vollständig kopiert
- [ ] Input-Ordner enthält alle Pflicht-Dokumente
- [ ] Content-Projektübersicht ist angelegt mit allen Produkten
- [ ] Produkt-Master-Sheet ist angelegt mit Basisdaten
- [ ] Ordner ist mit Kunden geteilt (Shared Drive)

### Asana-Projekt prüfen

- [ ] Projekt ist in Asana angelegt
- [ ] Alle Produkte sind als einzelne Tickets erfasst
- [ ] Jedes Ticket enthält:
  - [ ] Produktname + ASIN (falls vorhanden)
  - [ ] Link zum Input-Ordner
  - [ ] Link zur Content-Projektübersicht
  - [ ] Link zum Produkt-Master-Sheet
  - [ ] Zugewiesener Mitarbeiter
  - [ ] Fälligkeitsdatum (basierend auf SLA)
- [ ] Bei Retailer-Projekten: Timeline mit Meilensteinen erstellt

### Kommunikation

- [ ] Fulfillment-Mitarbeiter hat Kick-off mit Sales gehabt (oder Übergabe-Notiz erhalten)
- [ ] Kunden-E-Mail-Adresse ist bekannt und getestet
- [ ] Bei fehlendem Voice & Tone: Standard-Tone-Profil wird verwendet (dokumentieren welches)

---

## Eskalation bei unvollständigem Input

| Situation | Aktion | Verantwortlich | Frist |
|---|---|---|---|
| Pflicht-Input fehlt | Rückfrage an Kunden per E-Mail | Fulfillment-MA | Sofort |
| Keine Antwort nach 48h | Reminder-E-Mail + Sales informieren | Fulfillment-MA | 48h |
| Keine Antwort nach 5 Tagen | Eskalation an Projektleitung | Sales-MA | 5 Tage |
| Input unbrauchbar (z.B. falsche Sprache) | Rückfrage mit konkretem Beispiel | Fulfillment-MA | Sofort |

---

## Quick-Check: Ist der Kunde "production-ready"?

Schnelltest vor Produktionsstart:

| Check | Status |
|---|---|
| Kann ich das Produkt in einem Satz beschreiben? | Ja / Nein |
| Weiss ich, an wen sich das Produkt richtet? | Ja / Nein |
| Habe ich genug Info für eine Keyword-Recherche? | Ja / Nein |
| Weiss ich, wie die Marke kommuniziert? | Ja / Nein |
| Ist der Google Drive Ordner vollständig? | Ja / Nein |
| Ist das Asana-Ticket korrekt aufgesetzt? | Ja / Nein |

**Regel**: Mindestens 5 von 6 = "Ja" → Produktion starten. Sonst: Erst Input vervollständigen.
